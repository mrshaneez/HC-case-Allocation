Pilot build. Data saves automatically to this browser/device (it persists across reloads once deployed); use Export for a portable file, Share link to send a data snapshot, and Print / PDF for a filed copy of the matrix and assignments. Two boundaries to keep in mind: this is one register per device, not a single shared court-wide database, and the roles are a faithful model of admin control, not server-enforced security — so treat this as a pilot, not the system of record for live cases. The engine applies all four rules in order: recusal and the annual cap are hard limits; among eligible sections type rotation is the selector; and bench continuity breaks ties. Progressed cases keep their presiding judge only while under cap; a case that fits no eligible section is surfaced as unassignable. A shared multi-user database with real login is the next step.
